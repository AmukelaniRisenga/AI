1. What Was Built

Two fully functional AI-powered interactive prototypes hosted on Netlify, built in one week to demonstrate AI→3D pipeline design, multimodal reasoning, and interactive avatar animation — the two core product directions at NexEra.

Prototype 1 — 3D Asset Pipeline

User types a description ("yellow hard hat") or uploads a photo of a safety object. Claude Vision identifies it, maps it to a 3D model key, and renders a fully interactive 3D model in a Three.js viewer alongside an AI-generated educational safety summary.

Route: /prototype1

Prototype 2 — Avatar Animation

User types a natural language command ("wave hello to the learner"). Claude interprets intent, selects the best matching animation, and plays it on a 3D humanoid avatar with real skeletal-style motion — walk cycles, arm waves, head sweeps, safety postures.

Route: /prototype2

4. Prototype 1 — AI-Generated 3D Asset Pipeline

4.1 Claude Vision API — Object Identification

The backend receives either a text string or a base64-encoded image. It constructs a multimodal Claude message and constrains the output to a fixed enum of model keys to prevent hallucination and ensure every response maps to a known 3D asset.

// server/routes/analyzeObject.js — key logic const VALID_MODEL_KEYS = [ 'hard-hat', 'fire-extinguisher', 'wrench', 'hammer', 'safety-vest', 'drill', 'screwdriver', 'first-aid-kit', 'helmet', 'default-box' ] // Image path: multimodal content array content = [ { type: 'image', source: { type: 'base64', media_type: mimeType, data: imageBase64 } }, { type: 'text', text: `Identify the object. Respond ONLY with JSON: {"modelKey":"<key>","summary":"<2-3 sentence safety description>"} modelKey must be one of: ${VALID_MODEL_KEYS.join(', ')}` } ] // Text path: single string prompt content = `Given: "${text}" → respond ONLY with JSON: {"modelKey":"...","summary":"..."}` const message = await client.messages.create({ model: 'claude-sonnet-4-6', max_tokens: 512, messages: [{ role: 'user', content }] }) // Strip markdown fences Claude sometimes adds, then parse + validate const jsonStr = raw.replace(/^```json?\s*/i, '').replace(/```\s*$/i, '').trim() const parsed = JSON.parse(jsonStr) if (!VALID_MODEL_KEYS.includes(parsed.modelKey)) parsed.modelKey = 'default-box'
Critical design decision: Constraining Claude's output to a closed enum (rather than generating freeform model names) makes the pipeline deterministic and safe — any unrecognised object gracefully falls back to default-box instead of crashing the 3D loader.

4.2 Procedural 3D Model System

Since generating real GLB binary files programmatically is not practical, each model key maps to a distinct Three.js procedural geometry group. This approach actually shows deeper engineering ability than loading pre-made files.

// src/composables/useProceduralModels.js — hard-hat builder (example) const builders = { 'hard-hat'() { const g = new THREE.Group() // Yellow dome — top hemisphere only (phiLength = PI/2) const dome = new THREE.Mesh( new THREE.SphereGeometry(0.5, 20, 20, 0, Math.PI*2, 0, Math.PI/2), mat(0xffc107) // yellow ) g.add(dome) // Brim ring around the dome const brim = new THREE.Mesh( new THREE.TorusGeometry(0.58, 0.07, 8, 32), mat(0xffb300) // darker amber ) g.add(brim) return g }, // ... 9 more builders: fire-extinguisher, wrench, hammer, safety-vest, // drill, screwdriver, first-aid-kit, helmet, default-box } export function buildProceduralModel(scene, key) { const group = (builders[key] || builders['default-box'])() autoScaleCenter(group) // normalise to 2-unit bounding box, centre at origin scene.add(group) return group }

5. Prototype 2 — Natural Language → Avatar Animation

5.1 Claude NLP — Command → Animation Mapping

Claude interprets free-form English commands and maps them to one of 8 predefined animation keys. The prompt is engineered so Claude explains its mapping reasoning, which is shown to the user as educational context.

// server/routes/interpretCommand.js const VALID_ANIMATIONS = [ 'idle', 'wave', 'point', 'walk', 'safety-posture', 'crouch', 'thumbs-up', 'look-around' ] const prompt = `You are an AI animation controller for a 3D safety training avatar. Map this command to the best matching animation: "${command}" Available animations: ${VALID_ANIMATIONS.join(', ')} Respond ONLY with JSON: {"animationKey":"<animation>","explanation":"<1-2 sentences on safety relevance>"}` // After parsing, validate the key — never let an invalid key crash the viewer if (!VALID_ANIMATIONS.includes(parsed.animationKey)) parsed.animationKey = 'idle'

5.2 Procedural Skeletal-Style Animations

Instead of requiring pre-rigged GLB animation clips (Mixamo), the avatar uses pivot-group based procedural animation: each limb is a child of a pivot Group at the shoulder/hip joint, making rotations anatomically correct. A shared clock drives time-based trigonometric transforms every frame.

// src/components/proto2/AvatarViewer.vue — walk cycle example function animateBody(delta) { animTime += delta const t = animTime switch (currentAnim.value) { case 'walk': // Legs alternate at 3.2 Hz — opposite phase parts.leftLeg.rotation.x = Math.sin(t * 3.2) * 0.45 parts.rightLeg.rotation.x = -Math.sin(t * 3.2) * 0.45 // Arms counterswing (opposite to legs for realism) parts.leftArm.rotation.x = -Math.sin(t * 3.2) * 0.32 parts.rightArm.rotation.x = Math.sin(t * 3.2) * 0.32 // Body bounces at double frequency (both feet strike per cycle) parts.body.position.y = 1.0 + Math.abs(Math.sin(t * 6.4)) * 0.04 parts.body.rotation.y = Math.sin(t * 3.2) * 0.06 break case 'wave': // Right arm oscillates at 4.5 Hz with ±0.85 rad swing parts.rightArm.rotation.z = -0.3 + Math.sin(t * 4.5) * 0.85 parts.head.rotation.z = Math.sin(t * 4.5) * 0.08 break case 'look-around': // Head sweeps ±0.82 rad (≈47°) left/right; body follows at 17% amplitude parts.head.rotation.y = Math.sin(t * 1.4) * 0.82 parts.body.rotation.y = Math.sin(t * 1.4) * 0.14 break } }

5.3 Avatar Humanoid Structure

// Pivot groups give anatomically correct joint rotation origins const rightArmPivot = new THREE.Group() rightArmPivot.position.set(0.35, 1.42, 0) // ← at shoulder const rightArmMesh = new THREE.Mesh( new THREE.CapsuleGeometry(0.085, 0.5), bodyMat ) rightArmMesh.position.y = -0.3 // ← hangs down from pivot rightArmPivot.add(rightArmMesh) group.add(rightArmPivot) // To animate: rotate the pivot, not the mesh parts.rightArm = rightArmPivot
Key insight: Using pivot Groups (parent at joint, mesh as child offset below) means rotation.z = angle on the pivot naturally swings the arm around the shoulder — exactly how a real skeleton works. Without this, the arm would rotate around its own centre, looking broken.

6. Deployment Architecture

Local Development

  • npm run dev starts two processes via concurrently
  • Express API server on port 3001 (handles Claude API calls)
  • Quasar Vite dev server on port 9000
  • /api/* requests proxied from Quasar → Express (no CORS issues)

Production (Netlify)

  • quasar build → dist/spa/ served as static SPA
  • Netlify Functions mirror the Express routes (serverless Node)
  • netlify.toml redirects /api/* → /.netlify/functions/*
  • API key stored in Netlify environment variables (never in client bundle)

7. Key Engineering Decisions

Decision 1 — Procedural 3D over pre-made GLB assets

Challenge: No free, open-licensed GLB model library covers exactly the 10 safety objects needed, and downloading/converting binary assets at runtime is slow and unreliable.

Solution: Build each model from Three.js primitive geometry (SphereGeometry, CylinderGeometry, TorusGeometry, etc.) with colour-coded materials. Each model is visually distinct and immediately recognisable.

Benefit: Zero external dependencies for 3D content, instant render, easy to extend, and demonstrates deeper Three.js knowledge than simply loading a file.

Decision 2 — Closed enum validation on Claude outputs

Challenge: LLMs can hallucinate arbitrary strings. If Claude returns a model key that doesn't exist, the 3D loader silently fails or shows nothing.

Solution: Both API endpoints define a strict VALID_* array. After parsing Claude's JSON, any value not in the array is replaced with a safe default ('default-box' / 'idle'). The Claude prompt itself lists the exact allowed values.

Benefit: The pipeline never crashes on unexpected AI output. This is a production-grade pattern for constrained generation.

Decision 3 — Pivot-group skeleton for procedural avatar

Challenge: Three.js AnimationClip/KeyframeTrack works on named bones in skinned meshes. Building a proper rigged skeleton programmatically is complex and fragile.

Solution: Each limb is a child THREE.Group positioned at the joint (shoulder, hip). Rotating the Group rotates the attached arm/leg mesh around that joint — anatomically correct without a bone rig.

Benefit: Simple, readable animation code; easily extensible; no external animation files required.

Decision 4 — Dual backend (Express + Netlify Functions)

Challenge: Local development needs a real server for hot-reload and debugging. Production needs serverless for zero-cost hosting.

Solution: The Express route handlers are thin wrappers that can be imported by both the Express app and the Netlify function handlers. The same business logic runs in both environments.

Benefit: No code duplication; straightforward to add new endpoints; identical behaviour in dev and prod.

Decision 5 — Multimodal input (text + image) in Prototype 1

Challenge: Real training scenarios involve photographing physical equipment, not typing descriptions.

Solution: The frontend reads uploaded images via FileReader and encodes them as base64. The API endpoint switches between a text-only prompt and a multimodal [image, text] content array depending on which input is provided.

Benefit: Demonstrates the full Claude Vision capability — the same model that identifies a text description can identify a real photograph of a hard hat.

8. Challenges & How They Were Solved

Challenge What went wrong / Risk Solution
Claude markdown in JSON responses Claude occasionally wraps JSON in ```json fences despite explicit instructions not to Strip leading/trailing markdown fences with regex before JSON.parse(); never rely on Claude always following formatting instructions
Three.js canvas sizing Canvas doesn't auto-resize; aspect ratio breaks when panel resizes ResizeObserver on the container element calls renderer.setSize() + camera.updateProjectionMatrix() on every container size change
Model memory leaks Loading new 3D models without disposing old ones exhausts GPU memory useModelLoader.clearModel() traverses the old model and calls geometry.dispose() + material.dispose() before adding a new one. setCurrentModel() registers procedural models for the same cleanup path
Animation snap on transition Switching animations jumps abruptly because pose doesn't reset resetToNeutral() is called before every new animation; animTime resets to 0 so the new animation always starts from phase 0
Orbit controls on canvas vs DOM OrbitControls attached to document conflict with page scroll Pass the canvas element directly to new OrbitControls(camera, canvasRef.value) — events are scoped to the canvas only

9. How to Scale Inside NexEra's Platform

Prototype 1 Scaling Path

  • Real GLB assets — integrate Sketchfab API or Poly.pizza for on-demand asset retrieval; cache GLB files in CDN
  • Expand object library — add more model keys by adding one builder function; Claude prompt updates automatically
  • Streaming summaries — use Claude's streaming API to show the educational summary word-by-word as the model loads
  • Multi-object scenes — place several safety items in one scene for compound training scenarios
  • LMS embed — wrap the viewer in an iframe with a postMessage API for SCORM/xAPI integration

Prototype 2 Scaling Path

  • Mixamo GLB avatar — drop a rigged character with named animation clips into /public/avatar/character.glb; the useAnimationMixer composable handles clip playback immediately
  • Animation library expansion — add clips to the VALID_ANIMATIONS list and update Claude prompt; no other code changes needed
  • Voice input — Web Speech API → same interpretAndAnimate(command) store action
  • Multi-step sequences — Claude returns an array of animation keys; the mixer queues them with fadeOut/fadeIn transitions
  • Scene awareness — give Claude a scene manifest (objects present, their positions); it can issue contextually correct commands

10. Current Limitations

  • Procedural geometry only — models are geometric approximations; a production system needs real textured GLB assets from a 3D library or generation pipeline (e.g. Meshy.ai, Shap-E)
  • No skinned skeleton — avatar animations are procedural transforms on pivot groups, not true skeletal animation; adding a Mixamo-rigged GLB would give far more realistic motion
  • Fixed animation set — only 8 animation keys; complex compound actions ("pick up the extinguisher and carry it to the door") would need a sequence system
  • No persistent session — each API call is stateless; future work could add conversation history for contextual follow-up commands
  • No tests — unit and integration tests were not in scope for the one-week prototype
